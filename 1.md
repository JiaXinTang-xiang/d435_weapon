## 2. 环境启动（一条命令）

不用 `run.sh` 了，用 `uv` 管理环境（自动带 GPU）：

```bash
# 首次：安装依赖
cd /home/jiaxintang/桌面/model_wuqi/tip_detect_project
uv sync

# 每次运行
uv run python main.py
```

> 如果还没配好 `pyproject.toml`，先执行：
> ```bash
> uv init
> uv add ultralytics torch torchvision numpy opencv-python pyrealsense2 pyserial
> ```

## 3. 日常开发流程

### 3.1 每天开始 — 拉代码

```bash
cd /home/jiaxintang/桌面/model_wuqi/tip_detect_project
git pull origin main
```

### 3.2 写新功能 — 开分支

```bash
# 从 main 拉一条新分支
git checkout -b feature/你的功能名

# 写代码...

# 随时保存
git add .
git commit -m "做了什么"

# 推到 GitHub（备份 + 让队友看见）
git push origin feature/你的功能名
```

> **不要在 main 上直接改代码**，所有改动都在分支上做。

### 3.3 功能完成 — 合并

```bash
# 1. 先拉最新 main
git fetch origin
git checkout main
git pull origin main

# 2. 合并你的分支
git merge feature/你的功能名

# 3. 推上去
git push origin main

# 4. 删掉用完的分支（可选）
git branch -d feature/你的功能名
```
git push origin main --force
# 或更安全一点：
git push origin main --force-with-lease

## 6. 用 Pull Request（多人推荐）

### 你自己开发：
```bash
git checkout -b feature/xxx
# 写代码...
git push origin feature/xxx
# 去 GitHub → 点 "Compare & pull request" → 自己 Review → 合并
```

### 队友（小白）参与：
1. 小白 Fork 你的仓库到他自己 GitHub
2. 他在自己 Fork 里开发
3. 他提 PR 到你的仓库
4. 你来 Review 代码，没问题就合并

## 7. 常见问题处理

### 冲突了怎么办？

```bash
# git merge 提示 CONFLICT 时不慌

# 1. 看看哪些文件冲突了
git status

# 2. 打开冲突文件，找到 <<<<<<<  =======  >>>>>>> 
#    手动决定保留哪一边

# 3. 解决后提交
git add .
git commit -m "解决冲突"
git push origin main
```

### 改错了想回退

```bash
# 还没 commit：放弃修改
git checkout -- 文件名

# 已 commit 但没 push：
git reset HEAD~1

# 已经 push 了：用 revert（不改历史）
git revert HEAD
git push origin main
```

### 写了一半要切分支

```bash
# 暂存当前修改
git stash

# 干别的事...

# 回来恢复
git stash pop
```

## 8. 模型文件管理

`model/*.pt` 有 34MB，**不要直接存 Git**，建议用网盘共享。

或在 GitHub 仓库 Settings 里启用 Git LFS：

```bash
git lfs install
git lfs track "model/*.pt"
git add .gitattributes model/
git commit -m "Git LFS 管理模型"
git push origin main
```
